PROJET 01 - ARCHITECTURE RÉSEAU

Campus INTERCONTINENTAL

Conception et implémentation d'une infrastructure réseau VLSM avec FortiGate, Cisco L3 et Nextcloud

Version 1.0 - Janvier 2026
Équipe : 6 personnes
Durée : 8 semaines
Réseau / Sécurité / Cloud

Objectifs du projet

  • Déployer une infrastructure réseau hiérarchique pour le campus INTERCONTINENTAL (3 étages)
  • Implémenter un plan d'adressage VLSM optimisé avec la plage 172.17.0.0/16
  • Configurer un pare-feu FortiGate 7.0.3 pour la sécurisation et le routage inter-VLAN
  • Mettre en place un switch Cisco niveau 3 pour la commutation et le routage
  • Déployer un cloud privé Nextcloud pour le stockage étudiant
  • Assurer la segmentation réseau via VLANs (Admin, Invités, Étages, Serveurs)

Configuration des équipements

FortiGate 7.0.3

Interfaces VLAN

CLI FortiGate - Création VLANs 
config system interface
    edit "VLAN10_ADMIN"
        set vdom "root"
        set interface "internal"
        set type vlan
        set vlanid 10
        set ip 172.17.0.1 255.255.255.0
        set allowaccess ping
    next
    edit "VLAN20_INVITES"
        set interface "internal"
        set type vlan
        set vlanid 20
        set ip 172.17.1.1 255.255.255.0
    next
    edit "VLAN100_SERVERS"
        set interface "internal"
        set type vlan
        set vlanid 100
        set ip 172.17.16.1 255.255.255.0
        set allowaccess ping https ssh
    next
end

Règles de pare-feu

Politiques de sécurité 
config firewall policy
    edit 1
        set name "Classes_to_Internet"
        set srcintf "VLAN30_RDC" "VLAN40_ETAGE1" "VLAN50_ETAGE2"
        set dstintf "wan1"
        set srcaddr all
        set dstaddr all
        set action accept
        set service HTTP HTTPS DNS
        set nat enable
    next
    edit 2
        set name "Classes_to_Nextcloud"
        set srcintf "VLAN30_RDC" "VLAN40_ETAGE1" "VLAN50_ETAGE2"
        set dstintf "VLAN100_SERVERS"
        set dstaddr "Nextcloud_Server"
        set action accept
        set service HTTP HTTPS
    next
end

Déploiement Nextcloud sur Ubuntu Server 22.04

Installation des prérequis 
# Mise à jour du système
sudo apt update && sudo apt upgrade -y

# Installation LAMP
sudo apt install -y apache2 mariadb-server libapache2-mod-php
sudo apt install -y php-gd php-json php-mysql php-curl php-mbstring
sudo apt install -y php-intl php-imagick php-xml php-zip php-bz2
Configuration base de données 
sudo mysql -u root

CREATE DATABASE nextcloud;
CREATE USER 'nextcloud'@'localhost' IDENTIFIED BY 'MotDePasseFort123!';
GRANT ALL PRIVILEGES ON nextcloud.* TO 'nextcloud'@'localhost';
FLUSH PRIVILEGES;
EXIT;
Installation Nextcloud 
cd /var/www
sudo wget https://download.nextcloud.com/server/releases/latest.zip
sudo unzip latest.zip
sudo chown -R www-data:www-data nextcloud
sudo chmod -R 755 nextcloud
Configuration IP statique 
# /etc/netplan/00-installer-config.yaml
network:
  version: 2
  ethernets:
    eth0:
      addresses:
        - 172.17.16.10/24
      routes:
        - to: default
          via: 172.17.16.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4

Guide de dépannage

SymptômeCause probableSolution
Pas d'IP attribuéeDHCP non configuréVérifier service DHCP actif
Pas d'accès InternetRoute par défaut manquanteVérifier ip route 0.0.0.0
VLANs non routésip routing désactivéActiver ip routing sur switch L3
Nextcloud inaccessibleApache non démarrésudo systemctl restart apache2
Échec authentificationMauvais VLAN/portVérifier switchport access vlan

Tableau 2 : Guide de dépannage rapide

Problème : Les clients n'obtiennent pas d'IP du DHCP
Solution : Vérifier INTERFACESv4="eth0" dans /etc/default/isc-dhcp-server
Problème : Résolution DNS ne fonctionne pas
Solution : Configurer les forwarders dans /etc/bind/named.conf.options
