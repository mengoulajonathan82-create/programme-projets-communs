PROJET 01

Infrastructure réseau complète

Durée : 4 séances
Équipe : 6 personnes
Réalisé : Mars 2026

Objectifs du projet

  • Créer un réseau d'entreprise complet avec services essentiels
  • Configurer un serveur DHCP pour l'attribution automatique des adresses IP
  • Mettre en place un serveur DNS pour la résolution de noms
  • Déployer un serveur Web (Apache/Nginx) pour héberger un site
  • Installer un serveur de fichiers (Samba) pour le partage
  • Assurer la communication entre tous les services

Schéma de l'architecture

[Schéma réseau à insérer]

Routeur principal → Switch → Serveurs (DHCP, DNS, Web, Fichiers) + Postes clients

Étapes de réalisation

  1. Semaine 1 : Installation des serveurs (Ubuntu Server 22.04 LTS)
  2. Semaine 2 : Configuration du serveur DHCP (isc-dhcp-server)
  3. Semaine 3 : Mise en place du serveur DNS (Bind9)
  4. Semaine 4 : Déploiement du serveur Web (Nginx) et serveur de fichiers (Samba)

Commandes importantes

Configuration DHCP

# Installation du serveur DHCP
sudo apt update
sudo apt install isc-dhcp-server -y

# Configuration du fichier /etc/dhcp/dhcpd.conf
sudo nano /etc/dhcp/dhcpd.conf

# Exemple de configuration
subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.100 192.168.1.200;
    option routers 192.168.1.1;
    option domain-name-servers 8.8.8.8, 8.8.4.4;
    option domain-name "reseau.local";
    default-lease-time 600;
    max-lease-time 7200;
}

# Redémarrage du service
sudo systemctl restart isc-dhcp-server
sudo systemctl enable isc-dhcp-server

Configuration DNS (Bind9)

# Installation de Bind9
sudo apt install bind9 -y

# Configuration de la zone
sudo nano /etc/bind/named.conf.local

zone "reseau.local" {
    type master;
    file "/etc/bind/db.reseau.local";
};

# Création du fichier de zone
sudo cp /etc/bind/db.local /etc/bind/db.reseau.local
sudo nano /etc/bind/db.reseau.local

$TTL    604800
@       IN      SOA     ns.reseau.local. admin.reseau.local. (
                              2         ; Serial
                         604800         ; Refresh
                          86400         ; Retry
                        2419200         ; Expire
                         604800 )       ; Negative Cache TTL
;
@       IN      NS      ns.reseau.local.
@       IN      A       192.168.1.10
ns      IN      A       192.168.1.10
www     IN      A       192.168.1.20

# Redémarrage
sudo systemctl restart bind9

Difficultés rencontrées

  • Problème : Les clients n'obtenaient pas d'adresse IP du DHCP
  • Solution : Vérification des interfaces réseau dans /etc/default/isc-dhcp-server (ajout de INTERFACESv4="eth0")
  • Problème : Résolution DNS ne fonctionnait pas
  • Solution : Configuration des forwarders dans /etc/bind/named.conf.options

Résultats obtenus

DHCP fonctionnel : 15 clients connectés

DNS opérationnel : résolution interne et externe

Serveur Web : site accessible à http://www.reseau.local

Partage Samba : dossier partagé accessible
